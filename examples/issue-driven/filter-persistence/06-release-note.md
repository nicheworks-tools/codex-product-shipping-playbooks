# Release Note

## Summary

Selected filters are now preserved more reliably when returning to the listing page during normal in-app navigation.

## Audience

- End users

## What changed

Users who open an item detail page from the listing page and then return should now see their previously selected filters remain active more consistently.

## Why it matters

This reduces friction for users who browse filtered results and move back and forth between the listing page and item detail pages.

## Scope notes

- Included:
  filter preservation during the covered in-app return flow

- Not included:
  persistence across full browser reload
  broader search or filtering redesign

## Documentation impact

No major public documentation update is required for this example change.

## Recommended wording notes

Keep the wording narrow.
Do not imply that all possible navigation paths or all forms of persistence were redesigned.
